# Frapp UI assets — logos, icons, Open Graph

> Normative companion to [ui-brand-identity.md](ui-brand-identity.md). Defines **Frapp-owned** raster/SVG assets, where they live, and how apps consume them without drift.

---

## 1. Roles: Frapp vs chapter

| Asset kind | Owner | Purpose |
| ---------- | ----- | ------- |
| **Frapp app icon** (favicon / tab icon) | Product | Same mark on `frapp.live`, `app.frapp.live`, `docs.frapp.live` |
| **Frapp lockup** (icon + wordmark) | Product | Landing header, marketing email headers, future templates |
| **Open Graph image** | Product | Preview card when a **Frapp marketing URL** is shared (Slack, iMessage, etc.) |
| **Chapter logo** | Tenant | In-app surfaces, PDFs, onboarding — from Storage (`logo_path`); **does not** replace Frapp marks on marketing or docs |

Chapter accent color applies **inside chapter context** only per [ui-brand-identity.md](ui-brand-identity.md) §3.

---

## 2. Canonical package

All canonical files live in **`@repo/brand-assets`** (`packages/brand-assets/assets/`):

| File | Format | Use |
| ---- | ------ | --- |
| `app-icon.svg` | SVG 64×64 viewBox | Favicon / app icon; **source** for synced `app/icon.svg` |
| `frapp-lockup.svg` | SVG | Email embeds, download links, parity reference for inline React lockup |

**Do not** hand-edit `apps/*/app/icon.svg` — run `node scripts/sync-brand-assets.mjs` after changing `app-icon.svg`.

---

## 3. Dimensions and formats

| Surface | Spec |
| ------- | ---- |
| App icon | Rounded square mark; viewport 64×64; safe for 16px favicon scale |
| Lockup | Readable at ~120px width; word uses `fill="currentColor"` when inlined so `text-navy` / `dark:text-white` apply |
| Open Graph | 1200×630, generated by `apps/landing/app/opengraph-image.tsx` (aligned with navy `#0F172A` + accent `#60A5FA`) |

---

## 4. Next.js behavior

- **`app/icon.svg`:** App Router [file convention](https://nextjs.org/docs/app/api-reference/file-conventions/metadata/app-icons); emitted per deployment (immutable URL with build id).
- **`opengraph-image.tsx`:** [Open Graph image](https://nextjs.org/docs/app/api-reference/file-conventions/metadata/opengraph-image) route; avoids shipping a broken static `/og-image.png`.
- **Do not** set `metadata.openGraph.images` / `twitter.images` to static paths unless the file exists in `public/`.

---

## 5. Mobile (Expo)

Expo requires **raster** launcher icons. When `app-icon.svg` changes, regenerate `apps/mobile/assets/images/icon.png`, `adaptive-icon.png`, `favicon.png`, and splash assets per [docs/internal/BRAND_ASSETS.md](../docs/internal/BRAND_ASSETS.md).

---

## 6. Update procedure

1. Edit SVGs only under `packages/brand-assets/assets/`.
2. Run `node scripts/sync-brand-assets.mjs` from repo root.
3. Align `apps/landing/components/frapp-lockup.tsx` with `frapp-lockup.svg` if the lockup geometry changes (or regenerate from SVG).
4. Run `npm run check:brand-assets` (root) before PR.
5. Update [docs/internal/BRAND_ASSETS.md](../docs/internal/BRAND_ASSETS.md) if email paths or CI change.

---

## 7. Anti-patterns

- Duplicated “slightly different” icons per app.
- Chapter logo on the marketing homepage header.
- `og:image` pointing at a missing file (404 hurts crawlers and previews).
